Comment utiliser PecEngine

Release 1.0

Dernière révision le 10 janvier 2020

Ce logiciel est un logiciel Open Source.

PRESENTATION:

Ce logiciel a pour but de rassembler un ensemble d'outils utiles à l'élaboration de projets avec interface graphique.
Il est construit avec les seules fonctionnalités qu'offre Java SE 12 pour ne pas bousculer les habitudes de travail.
PecEngine permet de construire facilement une fenetre adapter à la réalisation de logiciels de bureaux ou de jeux videos.
Il permet de:

  1. Jouer des sons
  2. Gérer clavier et souris
  3. Adapter les composants à la taille de la fenetre
  4. Faire des sauvegarde sur fichier
  5. Créé des connection serveur/client
  6. Faire de l'injection de dépendance

Les différents chapitres:

  1. INSTALLATION
  2. FRAME
  3. PANEL
  4. SOUND
  5. LISTENERS
  6. INJECTOR
  7. FILES
  8. SERIALIZER
  9. SAVER

INSTALLATION:

Pour installer PecEngine, ajoutez le jar à votre projet et configurez le build-path.
Créer une classe qui sera votre main.

Elle doit extends de PecEngine:

public class Main extends PecEngine{ }

Puis ajoutez:

public static void main(String[] args){ PecEngine.start(new Main()); }

info: Frame hérite de JFrame et Panel hérite de JPanel

FRAME:

La classe Frame hérite de JFrame elle en a donc toutes les méthodes.
mais elle en possède aussi de nouvelles:

  1. Passer la frame en mode fenêtré si elle est en plein écran:
    frame.goWindowed();
  2. Passer la frame en plein écran si elle est en mode fenêtré:
    frame.goFullScreen();
  3. Active ou désactive le plein écran en fonction du mode dans lequel elle se trouve:
    frame.toggleFullScreen();
  4. Redéfinition de la méthode de JFrame pour quel fonctionne:
    frame.setSize(int width, int height);
  5. Fait simplement appel à la fonction setDisplayOption(int displayOption) de Panel:
    frame.setDisplayOption(int displayOption);

PANEL:

La classe Panel hérite de JPanel elle en a donc toutes les méthodes.
mais elle en possède aussi de nouvelles:

ARGUMENTS:

  1. Cet argument defini les options d'affichage
    private int displayOption = PROPORTIONAL;
  2. La taille, position et proportion des élements affichés est fixe:
    public static final int DEFAULT = 0;
  3. La taille, position et proportion des élements affichés s'adapte aux changements de taille de la fenêtre:
    public static final int PROPORTIONAL = 1;
  4. La taille, position des élements affichés s'adapte aux changements de taille de la fenêtre:
    public static final int ADAPTABLE = 2;

METHODES:

  1. Passer la frame en mode fenêtré si elle est en plein écran:
    panel.setDisplayOption(int displayOption);
  2. Passer la frame en plein écran si elle est en mode fenêtré:
    frame.goFullScreen();
  3. Active ou désactive le plein écran en fonction du mode dans lequel elle se trouve:
    frame.toggleFullScreen();
  4. Redéfinition de la méthode de JFrame pour quel fonctionne:
    frame.setSize(int width, int height);

SOUND:

Sound permet faire un outil destiner à jouer un son.
Il peut lire des fichiers au format AIFF, AU et WAV

Attention, seul les formats 8bit et 16bit sont supportés.
Cet outil utilise l'API javax.sound, il est possible d'en consulter la documentation: Java Sound Technology

Pour l'utiliser il faut simplement commencer par l'instancier: Sound sound = new Sound("son.wav"); Les methodes sont intuitives mais j'en fait la liste ici:

  1. Simplement jouer le son:
    sound.play();
  2. Mettre en pause:
    sound.pause()
  3. Relance la lecture après une pause:
    sound.restart()
  4. Stop la lecture:
    sound.stop()
  5. Déplace la lecture à la valeur en milliseconde indiqué:
    sound.jump(long time)
  6. Lit le son en boucle:
    sound.loop()
  7. Augmente le volume d'un cran:
    sound.upVolume()
  8. Baisse le volume d'un cran:
    sound.downVolume()
  9. Ajuste le volume à la valeur indiqué:
    sound.setVolume(double volume)

LISTENERS:

Une liste de listeners à été ajouter pour cibler les besoins.
Ils permettent des interaction plus simple avec les peripheriques clavier et souris.
Liste des listeners:

  1. KeyPressedListener
  2. KeyReleasedListener
  3. KeyTypedListener
  4. MouseClickedListener
  5. MouseDraggedListener
  6. MouseEnteredListener
  7. MouseExitedListener
  8. MouseMovedListener
  9. MousePressedListener
  10. MouseReleasedListener
  11. MouseWheelMovedListener

Il faut simplement implementer ces interfaces pour les utilisés, exemple:

public class Main implements PecEngine, MouseMovedListener{ }

Pensez à ajouter votre listener à la liste des listeners de votre frame, exemple:

@Override public void creation(Frame frame){ frame.addMouseMovedListener(this); }

INJECTOR:

Injector permet de faire facilement de l'injection de dependances.
Son utilisation ce passe en quatre étapes:

  1. La class à gérer doit implementer une interface quelconque:
    public class MaClass implements MonInterface{}
  2. Instancier un objet Injector, le fichier est créé automatiquement:
    Injector injector = new Injector("fichier.txt");
  3. Dans le fichier, inscrire le chemin de la classe séparer des autres par un espace ou un saut de ligne:
    MonPackage.MaClass
  4. Pour instencier MaClass, je fait:
    MonInterface monInterface = (MonInterface)injector.get("MonInterface");
  5. Il est possible d'appeler un constructeur avec de 0 à 5 arguments:
    MonInterface monInterface = (MonInterface)injector.get("MonInterface", arg1, ag2, arg3);
  6. Il est possible d'appeler un constructeur avec de 0 à plus de 5 arguments:
    MonInterface monInterface = (MonInterface)injector.get("MonInterface", new Object[]{arg1, ag2, arg3});

FILES:

Permet de lire et écrire dans un fichier:

  1. Construit et retourne le chemin complet du fichier name si il se trouve dans le dossier courant:
    Files.loadPath(String name);
  2. Construit et retourne un implementation de File:
    Files.loadFile(String name);
  3. Construit et retourne un implementation de BufferedImage:
    Files.loadBufferedImage(String name);
  4. Construit et retourne un implementation de FileReader:
    Files.loadFileReader(String name);
  5. Construit et retourne un implementation de FileWriter:
    Files.loadFileWriter(String name);
  6. Construit et retourne un implementation de FileWriter:
    Files.loadFileWriter(String name, boolean b);
  7. Construit et retourne un implementation de BufferedReader:
    Files.loadBufferedReader(String name);
  8. Construit et retourne un implementation de BufferedWriter:
    Files.loadBufferedWriter(String name);
  9. Construit et retourne un implementation de BufferedWriter:
    Files.loadBufferedWriter(String name, boolean b);

  10. Files.loadScanner(String name);

SERIALIZER:

Permet de sérialiser et déserialiser des objets qui implementent l'interface Serializable:

  1. Transforme un objet Serializable en tableau de byte
    Serializer.serialize(Serializable exchange);
  2. Transforme un tableau de byte en objet Serializable
    Serializer.deserialize(byte[] byteTab);
  3. Transforme un objet Serializable en String
    Serializer.serializeString(Serializable exchange);
  4. Transforme un String en objet Serializable
    Serializer.deserializeString(String byteTab);

SAVER:

Utilise Files et Serializer pour sauvgarder des objets dans un fichier:

  1. Dans un premier temps vous devez créé une instance de la classe Saver:
    Saver saver = new Saver("file.txt");
  2. Sauvegarde par lignes dans un fichier, les élements d'une liste d'objets qui implemente l'inteface Serializable
    saver.save(List<Serializable> list);
  3. Lit un fichier et retourne la liste des lignes qu'il contient sous forme d'objets de type Serializable
    saver.load();
  4. Sauvegarde par lignes dans un fichier, les élements d'une liste de String
    saver.saveStrings(List<String> list);
  5. Lit un fichier et retourne la liste des lignes qu'il contient
    saver.loadStrings();
